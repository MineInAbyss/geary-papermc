# Changelog

All notable changes to this project between Minecraft versions will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/). We currently do not follow semver while
MAJOR is set to zero.

## [Unreleased]

### Added

- `GearyPlugin` interface now provides nicer access for registering new `WorldScoped` instances for Geary. In the future
  this will be expanded to create scopes whenever a new Geary world is created (i.e. in a Folia style system where each
  region gets its own world.)

### Changed

- Refactored our feature system to use our new [DI library](https://github.com/MineInAbyss/dependencies-kt). Plugins
  depending on geary-papermc now get a nicer entrypoint via `gearyPaper.configure { ... }`.
- Refactored internals for Idofront 2.0.0 and Geary 0.29.0, namely cleaner config reading, register a
  SerializableItemStack prefab provider.
- Moved MultiEntryYamlReader to Idofront (via its configs module)
- Switched to a .sql file with bindings automatically generated by sqlite-kt for the spread spawning system
- Use single table for spread spawns instead of per world, optimize to use a `spawnCategory` column in rtree rather than
  joining (which would previously cause a slow database SCAN)

### Fixed

- **Geary reloads** should now be much more stable, our new DI system gives `AutoCloseable` scopes that automatically
  unregister systems when reloading, hopefully fixing issues like item observers being called twice after a reload.
- Skip already registered recipes to prevent users from crashing on reloads, since Bukkit sends ALL recipes for each
  modified recipe.
